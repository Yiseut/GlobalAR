# Topic 页面改造规范 v2

## 一、设计目标

把 topic.html 从"数据完整度审计页"改造成"行业格局叙事页"。
读者打开第一屏 3 秒内能回答：**这个赛道有多大、谁是头部、动能在哪、近期有什么新东西**。

所有 6 个子赛道页 (injectables / ebd / regenerative / botulinum / threads / mesotherapy) 复用同一骨架，只换数据切片。

---

## 二、五层叙事骨架（按从上到下顺序）

### L1 概览叙事（首屏，必须完整出现）

替换现有的 hero + 8 个 KPI 块。新结构：

- **Hero 标题区**：保留，但副标题增加一句"格局速读"，由数据驱动自动生成，例如：
  "全球 N 家企业、M 款在售产品，CR5 集中度 X%，过去 12 个月新增 Y 款获批。"

- **6 个核心 KPI**（替换现有 8 个，全部要求带"变化"指标）：
  1. 在售产品数（+ 近 12 个月新增 N）
  2. 全球企业数（+ Top5 集中度 CR5）
  3. 监管市场覆盖数（+ 覆盖区域数）
  4. 在研管线数（+ 同比）
  5. 适应症覆盖宽度（+ 新增适应症数）
  6. 跨境流通指数（本土品牌 / 进口品牌比）

每张 KPI 卡保留主数值 + 一行 delta（同比 / 环比 / 占比）+ 一个 sparkline 缩略图（12 个月）。
**移除**：产品/企业/品牌/国家/子赛道/适应症/区域/监管 这种纯计数堆砌。

### L2 竞争格局

替换"业务角色 / 公司属性 / 技术类型"3 个 bar-list 模块。新结构：

- **A. 集中度卡**：一张 donut chart 显示 CR5 / CR10 / 其余尾部的占比，中间数字写 HHI。
- **B. 头部公司 Top10**：横向条形图，长度=产品数，条形旁边显示国家旗 + 子赛道徽章（最多 3 个），点击钻取该公司产品。
- **C. 价值链位置堆叠图**：把每家公司归类到（原料 / 制造 / 品牌 / 分销 / 服务）一个或多个环节，画成 100% 堆叠柱。
- **D. 本土 vs 进口对比**：按当前区域（默认全球）拆，简单对比柱。
- **E. 子赛道生命周期矩阵**：散点图，X = 累计获批数（规模），Y = 近 3 年获批 / 累计获批（成长率），气泡大小 = 企业数。把每个子赛道标在四象限。

### L3 动能与时间

新增一整段，当前页面完全缺失。

- **A. 准入时间线**：按年的堆叠柱（区分子赛道颜色）+ 累积曲线叠加。X 轴年份，Y 轴当年获批数。
- **B. 近 12 个月新进入者**：卡片 list，每张卡 = 公司名 + 产品名 + 国家 + 获批日期 + 适应症标签。
- **C. 在研管线 vs 在售**：双柱并排，按子赛道分组。
- **D. 监管事件流**：竖向时间线，显示重大获批 / 召回 / 警告 / 退市，每条一行。

### L4 深挖矩阵

替换现有 5 张热力图，收敛到 3 张，**重要：单元格颜色编码"数量"而非"有/无"**：

- **A. 国家 × 子赛道**：单元格颜色深浅 = 该国该子赛道的产品数。
- **B. 适应症 × 区域**：单元格 = 官方获批数（不是定位标签）。
- **C. 公司 × 价值链**：单元格 = 该公司在该环节的产品 / 业务条数。

每张矩阵右上角放一个"洞察气泡"，用一句话总结最显眼的格局（例如"巴西在中胚层产品数全球第一"），由数据驱动自动生成。
单元格点击 → 弹层显示底下的产品列表。

### L5 产品明细

保留现有 records-table，但增加：
- 列筛选下拉（按子赛道 / 国家 / 价值链位置）
- 导出 CSV 按钮
- 按子赛道折叠分组
- 表头加排序

## 三、从主流程移除的模块

以下模块**不要删除代码**，迁移到一个新的 `/audit.html` 页面：

- "本专题证据状态"（topicEvidenceFunnel）
- "本专题监管线索 已合并 / 候选"（topicRegulatoryMix）
- "产品级证据完整度"（topicProductEvidenceAudit）

在主 topic.html 顶部右上角加一个小徽章：「数据完整度 86% · 查看审计」 链接到 audit.html。

---

## 四、视觉一致性规则

- **颜色**：每个子赛道分配一个固定主题色，整页所有图表里这个子赛道的颜色都一致。颜色映射写在 app-data.js 里作为常量。
- **形式多样化**：现有页面 7 个 bar-list 看着累。新设计里 bar-list 只允许出现在 L2-B 一处。其他位置改用 donut / stacked bar / scatter / timeline / sparkline。
- **每个图右上角必须有"洞察气泡"**：一句不超过 25 字的中文 + 英文短句，由数据驱动。
- **空状态**：所有图表加 empty state 文案，不要直接显示 0 或空白。

---

## 五、交互规范

- 所有图表元素点击都应该钻取到产品明细表（自动应用筛选）
- 顶部专题导航当前选中态高亮
- KPI 卡的 sparkline hover 显示具体数值
- 矩阵单元格 hover 显示 tooltip（含数量 + 1 行样例公司）
